## 🎯 核心流程

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        SDD 完整开发流程                                  │
└─────────────────────────────────────────────────────────────────────────┘

用户描述需求
    │
    ↓
┌─────────────────────────────────────────────────────────────────────────┐
│  1. 意图识别（NLP Parser）                                              │
│     • 分析自然语言需求                                                  │
│     • 提取关键信息                                                      │
│     • 识别技术栈                                                        │
└─────────────────────────────────────────────────────────────────────────┘
    │
    ↓
┌─────────────────────────────────────────────────────────────────────────┐
│  2. 需求澄清（Clarifier）                                               │
│     • 提出澄清问题（选择题）                                            │
│     • 确认业务规则                                                      │
│     • 确认验收标准                                                      │
└─────────────────────────────────────────────────────────────────────────┘
    │
    ↓
┌─────────────────────────────────────────────────────────────────────────┐
│  3. Spec 生成（Spec Generator）                                         │
│     • 生成结构化 Spec 文档                                               │
│     • 包含需求/设计/任务拆分/验收标准                                   │
│     • 支持流式输出                                                      │
└─────────────────────────────────────────────────────────────────────────┘
    │
    ↓
┌─────────────────────────────────────────────────────────────────────────┐
│  4. Spec 审核（User Review）                                            │
│     • 用户查看 Spec                                                     │
│     • 通过对话修改 Spec                                                 │
│     • 确认后点击运行                                                    │
└─────────────────────────────────────────────────────────────────────────┘
    │
    ↓
┌─────────────────────────────────────────────────────────────────────────┐
│  5. 任务规划（Task Planner）                                            │
│     • 拆分任务为多个子任务                                              │
│     • 生成 To-do List                                                   │
│     • 规划执行顺序                                                      │
└─────────────────────────────────────────────────────────────────────────┘
    │
    ↓
┌─────────────────────────────────────────────────────────────────────────┐
│  6. 代码生成（Code Generator）                                          │
│     • 调用 LLM 模型生成代码                                              │
│     • 多文件同时生成                                                    │
│     • 添加注释和文档                                                    │
└─────────────────────────────────────────────────────────────────────────┘
    │
    ↓
┌─────────────────────────────────────────────────────────────────────────┐
│  7. 质量验证（Quality Check）                                           │
│     • 代码规范检查                                                      │
│     • 生成并运行测试                                                    │
│     • 检查测试覆盖率                                                    │
│     • 自动修复问题                                                      │
└─────────────────────────────────────────────────────────────────────────┘
    │
    ↓
┌─────────────────────────────────────────────────────────────────────────┐
│  8. 验收结果（User Acceptance）                                         │
│     • 用户查看生成的代码                                                │
│     • 查看测试结果                                                      │
│     • 决定 Accept/Reject                                                │
└─────────────────────────────────────────────────────────────────────────┘
    │
    ↓
完成！代码已应用到工作区
```
