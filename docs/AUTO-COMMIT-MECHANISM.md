# 自动提交和自我修复机制

> 完整的持续提交流程设计

**创建时间：** 2026-04-01  
**维护者：** 池少团队

---

## 🎯 设计目标

1. **持续检测** - 每 5 分钟自动检查
2. **自动提交** - 有变更自动提交
3. **自我修复** - 失败自动修复
4. **实时通知** - 创建通知文件提示

---

## 🏗️ 系统架构

```
┌─────────────────────────────────────────┐
│     Continuous Monitor (持续监控)       │
│  - 每 5 分钟检查一次                     │
│  - 检查文档完整性                        │
│  - 检查文档质量                          │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│     Auto-Commit System (自动提交)       │
│  - 检测变更                              │
│  - Git 提交                              │
│  - Git 推送                              │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│     Self-Heal (自我修复)                │
│  - Git 配置修复                          │
│  - 远程仓库检查                          │
│  - 代码拉取                              │
└─────────────────────────────────────────┘
```

## 📁 核心脚本

### 1. auto-commit-system.sh

**功能：** 自动提交流程

**特性：**
- ✅ 检测变更
- ✅ 自动提交（带时间戳）
- ✅ 自动推送
- ✅ 失败重试（最多 3 次）
- ✅ 自我修复
- ✅ 通知文件

**使用：**
```bash
bash auto-commit-system.sh
```

---

### 2. continuous-monitor.sh

**功能：** 持续监控

**特性：**
- ✅ 每 5 分钟检查一次
- ✅ 检查文档完整性
- ✅ 检查文档质量
- ✅ 触发自动提交
- ✅ 显示通知

**使用：**
```bash
bash continuous-monitor.sh
```

---

### 3. start-services.sh

**功能：** 启动所有服务

**特性：**
- ✅ 停止旧进程
- ✅ 启动监控服务
- ✅ 立即执行一次提交
- ✅ 显示状态和日志路径

**使用：**
```bash
bash start-services.sh
```

## 🔄 工作流程

### 正常流程

```
1. 监控服务启动
   ↓
2. 每 5 分钟检查
   ↓
3. 检测变更
   ↓
4. Git 提交
   ↓
5. Git 推送
   ↓
6. 等待下次检查
```

### 异常流程

```
1. 提交失败
   ↓
2. 重试（最多 3 次）
   ↓
3. 仍然失败
   ↓
4. 启动自我修复
   ↓
5. 修复后重试
   ↓
6. 创建通知文件
```

## 📊 日志文件

| 文件 | 用途 | 查看命令 |
|------|------|----------|
| logs/monitor.log | 监控日志 | `tail -f logs/monitor.log` |
| logs/auto-commit.log | 提交流程 | `tail -f logs/auto-commit.log` |
| logs/pending-commits.txt | 待处理通知 | `cat logs/pending-commits.txt` |
| logs/monitor.pid | 监控进程 PID | `cat logs/monitor.pid` |

## 🔧 自我修复机制

### 修复项

1. **Git 配置**
   - 自动设置 user.name
   - 自动设置 user.email

2. **远程仓库**
   - 检查 origin 是否存在
   - 不存在则添加

3. **代码同步**
   - 自动 pull 最新代码
   - 解决冲突

## 📢 通知机制

### 通知类型

| 类型 | 说明 | 示例 |
|------|------|------|
| ✅ 提交成功 | Git 提交成功 | ✅ 提交成功：auto: 持续迭代提交... |
| ✅ 推送成功 | GitHub 推送成功 | ✅ 推送成功 |
| ❌ 推送失败 | GitHub 推送失败 | ❌ 推送失败，下次重试 |
| ❌ 需要介入 | 需要人工处理 | ❌ 提交失败，需要人工介入 |

### 查看通知

```bash
# 查看待处理通知
cat logs/pending-commits.txt

# 监控通知（实时）
tail -f logs/pending-commits.txt
```

---

## 🚀 启动和停止

### 启动服务

```bash
bash start-services.sh
```

### 停止服务

```bash
# 停止监控进程
pkill -f "continuous-monitor.sh"

# 或者使用 PID
kill $(cat logs/monitor.pid)
```

### 查看状态

```bash
# 查看进程
ps aux | grep monitor

# 查看日志
tail -f logs/monitor.log
```
