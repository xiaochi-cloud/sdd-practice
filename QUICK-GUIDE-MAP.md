## 🎯 关键要点

### 1. 核心流程

```
描述需求 → Quest 生成 Spec → 审核 Spec → 自主编程 → 验收结果
```

### 2. Spec 格式

**自然语言（推荐）：**
```
创建一个用户注册功能，包含：
- 用户名、邮箱、密码
- 密码强度验证
- 用户名唯一性检查
```

**OpenSpec YAML（复杂功能）：**
```yaml
name: UserService
functions:
  - name: createUser
    parameters:
      - name: username
        type: string
```

### 3. 执行环境

| 环境 | 适合 | 特点 |
|------|------|------|
| Local | 简单任务 | 快速反馈 |
| Parallel | 中等复杂 | 隔离安全 |
| Remote | 复杂长程 | 云端执行 |

### 4. 验收标准

- ✅ 代码生成完成
- ✅ 测试覆盖率 80%+
- ✅ 编译通过
- ✅ 所有测试通过
