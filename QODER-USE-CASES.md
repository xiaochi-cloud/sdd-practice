# Qoder 使用场景拆解

> 面向 Java 开发者的 Qoder 实战指南

**版本：** v1.0 | **日期：** 2026-03-12

---

## 🎯 Qoder 能做什么

### 核心能力（Quest Mode）

```
1. Spec 驱动开发
   - 描述需求 → 生成 Spec → 自主编程 → 验收结果
   
2. 搭建网站
   - 描述想法 → 自动搭建 → 实时预览
   
3. 原型探索
   - 快速验证 → 迭代优化
```

## 📋 使用场景清单

### 场景 1：开发新功能 ⭐⭐⭐⭐⭐

**适合：** 业务功能开发、CRUD 操作

**示例：**
```
"创建一个用户注册功能，包含：
- 用户名、邮箱、密码输入
- 密码强度验证（大小写 + 数字）
- 用户名唯一性检查
- 邮箱唯一性检查
- 注册成功后自动登录

技术栈：
- Spring Boot 3
- MySQL
- BCrypt 密码加密
- JWT Token

验收标准：
- 有效输入能成功注册
- 用户名重复返回 409
- 邮箱重复返回 409
- 密码强度不足返回 400"
```

**演示价值：** ⭐⭐⭐⭐⭐
- 展示 Spec 驱动流程
- 展示多文件修改
- 展示业务规则表达

## 🎯 明天演示推荐

### 推荐演示：用户注册功能 ⭐⭐⭐⭐⭐

**为什么选择这个：**

1. **贴近业务** - 每个系统都有用户注册
2. **逻辑清晰** - 输入→验证→保存→返回
3. **展示全面** - 后端 + 测试 + 文档
4. **时间合适** - 10-15 分钟完成

---

### 演示流程（15 分钟）

#### 步骤 1：打开 Qoder IDE（1 分钟）

```
1. 打开 Qoder IDE
2. 打开示例项目
3. 切换到 Quest Mode
```

**话术：**
```
"现在我打开 Qoder IDE，切换到 Quest Mode。
这是 Quest Mode 的界面，左边是任务列表，
中间是对话区，右边是产物区。"
```

---

#### 步骤 2：创建任务（2 分钟）

```
1. 点击 New Quest
2. 选择"Spec 驱动"场景
3. 输入任务描述
```

**输入内容：**
```
创建一个用户注册功能：

功能描述：
- 用户可以通过邮箱和密码注册账号
- 注册成功后自动登录

输入：
- username: 用户名（3-20 字符，只能包含字母、数字、下划线）
- email: 邮箱地址（有效邮箱格式）
- password: 密码（8-32 字符，必须包含大小写字母和数字）

输出：
- user_id: 用户 ID（UUID）
- token: JWT 访问令牌

业务规则：
- 用户名不能重复
- 邮箱不能重复
- 密码必须加密存储（BCrypt）
- 注册成功后生成 JWT Token（有效期 24 小时）

技术栈：
- Spring Boot 3
- MySQL
- BCrypt
- JWT

验收标准：
- 有效输入能成功注册，返回 201
- 用户名重复返回 409
- 邮箱重复返回 409
- 密码强度不足返回 400
- 用户名格式错误返回 400
```

**话术：**
```
"现在我描述需求。注意看，我用自然语言描述，
不需要学复杂的格式。

我定义了：
- 输入：用户名、邮箱、密码
- 输出：用户 ID、Token
- 业务规则：唯一性、密码加密
- 验收标准：5 个测试用例

Qoder 会基于这个生成 Spec 文档。"
```

---

#### 步骤 3：审核 Spec（3 分钟）

```
1. Quest 生成 Spec 文档
2. 查看 Spec 内容
3. 确认无误
```

**话术：**
```
"Qoder 生成了 Spec 文档。我们来看一下：

- 功能描述准确
- 输入输出完整
- 业务规则清晰
- 验收标准覆盖

如果有问题，可以直接对话修改。
比如'把密码长度改成 6-32 字符'。

确认无误后，点击'运行 Spec'。"
```

---

#### 步骤 4：执行代码（5 分钟）

```
1. Quest 自主编程
2. 查看进度（To-do List）
3. 查看生成的代码
```

**话术：**
```
"现在 Qoder 在自主编程。看这边，
有实时的 To-do List，显示进度。

- 分析需求 ✓
- 设计数据模型 ✓
- 生成实体类 ✓
- 生成 Service ✓
- 生成 Controller ✓
- 生成测试用例 ✓

大概 3-5 分钟完成。
完成后会提示验收。"
```

---

#### 步骤 5：验收结果（3 分钟）

```
1. 查看生成的代码
2. 查看测试用例
3. 运行测试
4. Accept 应用修改
```

**话术：**
```
"完成了！我们来看一下生成的代码：

- UserService.java - 接口定义
- UserServiceImpl.java - 实现类
- UserController.java - REST 接口
- User.java - 实体类
- UserServiceTest.java - 单元测试

测试覆盖率 85%，所有测试通过。

确认无误后，点击'Accept'应用修改。"
```
