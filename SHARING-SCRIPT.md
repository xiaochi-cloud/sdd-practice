### 【核心概念】（8 分钟）

```
先问一个问题：什么是 SDD？

SDD = Specification-Driven Development
规范驱动开发

听起来很复杂？其实很简单。

SDD 中的"规范"，就像：

1. Java 的 Interface
   （展示代码对比）
   
   Java Interface:
   public interface UserService {
       RegisterResponse register(RegisterRequest request);
   }
   
   OpenSpec 规范:
   features:
     - name: 用户注册
       inputs:
         - name: request
           type: RegisterRequest

   是不是一样？都是定义契约！

2. 建筑的施工图
   （展示图片）
   工人按施工图建房，
   我们按规范写代码。

3. 乐谱
   乐手按乐谱演奏，
   AI 按规范生成代码。

核心价值有 4 个：

1. 减少需求理解偏差 ↓70%
2. 文档永远同步
3. AI 协作更顺畅
4. 团队沟通成本 ↓30%
```

### 【快速演示】（5 分钟）

```
现在，我花 5 分钟，演示 5 分钟能做什么。

（切换到终端）

第 1 分钟：安装
pip install qoder-cli
qoder --version

第 2 分钟：初始化
mkdir demo
cd demo
qoder init --lang java

第 3-4 分钟：编写规范
（打开编辑器，编写简单规范）

features:
  - name: 打招呼
    inputs:
      - name: name
        type: string

第 5 分钟：生成代码
qoder generate --all
mvn test

看到了吗？
5 分钟，从 0 到可运行代码！

传统方式：至少 30 分钟
SDD 方式：5 分钟
效率提升 6 倍！
```
